# 方案：即时零售成本与财务一致性（对照《流程说明》）

## 背景摘要

平台爬虫账单缺少活动 ID，成本需由运营按月拆到项目；实际结算滞后，若不在制度上约束「关账 + 当期调整」，易出现财务库存与真实结算偏离。

## 目标原则

1. **强校验依据**：以业务账单、Excel 补贴合计与关联成本认领合计的一致性作为提交前置条件（已有）。
2. **禁止跨月回溯**：已推送财务的归集行锁定，不允许在来源月改数；差错在**当前财务入账月**通过调整分录体现。
3. **结算后不漏调整**：业务账单「已同步」后自动比对「结算项目 vs 成本归集主项目」，生成待办并留痕。
4. **全链路可查**：关联账单、账单同步触发的核对、操作人/时间/财务归属月写入审计列表。

## 前端原型已实现能力

| 能力 | 说明 |
|------|------|
| 财务状态列 | 成本行 `financeLineStatus: locked` 时展示「财务已锁定」，隐藏「关联账单」，提供「锁定说明」。 |
| 结算核对页 | 侧栏「结算核对」：`reconciliationTasks` 待办 + `costChangeAudit` 留痕表。 |
| 账单提交后 | `runBillSyncReconciliation` 追加项目不一致 / 缺归集项目类待办。 |
| 账单详情 | 「项目一致性核对」卡片：逐条关联成本 vs 账单项目。 |
| 分配记录「调整」 | 弹窗明确「当期调整」路径并跳转结算核对。 |

## 后续接真实系统时的建议

- 财务关账、调整单与 PMS 库存扣减由后端强一致事务驱动；前端状态与只读策略与接口字段对齐。
- `reconciliationTasks` 与财务凭证号、结算单号双向引用，支持关闭待办时上传依据附件。
