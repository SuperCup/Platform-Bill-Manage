# 用户足迹 — Platform-Bill-Manage

## 2026-05-09

### 需求：三系统前端交互设计

**用户输入：**
设计三个系统（仅前端交互），分别是：

1. **到店营销** — 微信、支付宝、抖音等平台的活动运营管理系统
   - 提供业务数据支持，需要与客户结算时从该系统归集账单
   - 基于账单按月汇总财务成本
   - 获取微信优惠券、微信小店、支付宝优惠券、支付宝碰一下、抖音团购对应账单
   - 支持每月1号自动归集上月成本，也支持运营人员按需手动归集
   - 归集基于活动账单，不可有重叠
   - 当活动关联项目发生变更时，需要有账单归集引用的校验

2. **即时零售** — 美团闪购、淘宝闪购、京东到家、多点等平台的活动运营管理系统
   - 账单中无活动 ID，无法直接与活动绑定，作用与到店营销相同
   - 获取对应平台账单文件（通过爬虫，已实现）
   - 按月份 + 客户 + 二级实体（平台同账号下多个实体）生成成本记录
   - 每月1号由运营人员分配（输入）成本归属项目金额
   - 需要有实际成本与已认领成本明细
   - 分配时系统自动拆分已入库账单明细，拼凑对应分配金额
   - 如明细金额不够，允许运营人员编辑部分记录
   - 分配后运营人员可再次调整，并归集

3. **PMS** — 项目管理系统
   - 以上两个系统管理活动依赖的项目来源
   - 管理项目报价、结算、开票等
   - 汇总每月业务系统的成本，用作库存监控

**输出要求：**
- 系统通信架构图
- 几个系统必要模块的功能页面
- 风格参照图片中组件样式（中式企业管理系统，白色主题，蓝色 accent）
- 交互方式简洁直观

#### 补充：Windows 上 `claude` 报需要 Git bash 或 PowerShell 7

**现象：** `claude -v` 正常，但运行 `claude` 提示需要 Git for Windows 或 PowerShell 7，或设置 `CLAUDE_CODE_GIT_BASH_PATH`。

**原因：** 默认使用 **Windows PowerShell 5** 时，Claude Code 需要 **Git 自带的 bash** 或 **PowerShell 7（`pwsh`）**；仅安装 Git 且 PATH 里只有 `Git\cmd` 时，`bash.exe` 仍可能找不到。

**本机已做：** 通过 winget 安装 `Git.Git`（2.54.0），并将 `C:\Program Files\Git\bin` 追加到**当前用户**的 PATH，使 `where bash` 可解析到 `bash.exe`。

**请你操作：** **完全关闭并重新打开** PowerShell（或 Cursor 内置终端），再执行 `where bash` 与 `claude`。

**备选（任选其一）：**

- 安装 PowerShell 7：`winget install Microsoft.PowerShell`
- 手动指定 bash：用户或系统环境变量 `CLAUDE_CODE_GIT_BASH_PATH` = `C:\Program Files\Git\bin\bash.exe`
